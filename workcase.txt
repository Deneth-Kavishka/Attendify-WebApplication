## 🏗️ SYSTEM OVERVIEW

**Project Name:** SmartTrack Attendify
**Version:** 2.0 (Updated 2025)
**Technology Stack:** React + Node.js + Python + Arduino + Firebase + PostgreSQL

### What This System Does:
- ✅ **Face Recognition Attendance** - ESP32-CAM captures student faces automatically
- ✅ **RFID Card Backup** - NodeMCU with RFID reader for alternative authentication
- ✅ **Multi-Role Dashboard** - Admin, Lecturer, and Student portals
- ✅ **Real-time Updates** - Live attendance feed and hardware status monitoring
- ✅ **Automatic Eligibility** - Calculate exam eligibility based on attendance %
- ✅ **Offline Support** - Hardware works offline and syncs when online

## 🔧 TECHNICAL ARCHITECTURE

### Frontend (Client)
- **Framework:** React 18 + TypeScript + Vite
- **UI Library:** Shadcn/UI + Radix UI + Tailwind CSS
- **State Management:** TanStack Query + React Context
- **Routing:** Wouter (lightweight router)
- **Real-time:** WebSocket + Firebase listeners

### Backend (Server)
- **Runtime:** Node.js + Express.js + TypeScript
- **Database:** PostgreSQL + Drizzle ORM (currently using in-memory for demo)
- **Authentication:** Token-based (JWT ready)
- **API:** RESTful endpoints with role-based access
- **Real-time:** WebSocket server

### AI Service (Python)
- **Framework:** Flask + CORS
- **Computer Vision:** OpenCV + face_recognition + dlib
- **Face Processing:** Encoding, detection, matching (85% confidence threshold)
- **Storage:** Local pickle files + Firebase backup
- **Port:** 5001

### Hardware Components
- **ESP32-CAM:** Face capture and recognition trigger
- **NodeMCU + RFID:** Card-based attendance backup
- **Communication:** WiFi + HTTP REST API
- **Display:** LCD screen for feedback
- **Audio:** Buzzer for confirmation

### Database Schema
- **Users** - Authentication and profiles
- **Students** - Student data with RFID cards and face embeddings  
- **Lecturers** - Instructor profiles and specializations
- **Classes** - Course management with schedules
- **Attendance Records** - Face recognition and RFID data
- **Hardware Devices** - ESP32-CAM and RFID device management
- **Exam Eligibility** - Automated eligibility calculations

## 🚀 HOW THE SYSTEM WORKS - COMPLETE WORKFLOW

### 1. SYSTEM STARTUP SEQUENCE

**Backend Initialization:**
```
1. Node.js server starts on port 5000
2. Demo data initialized (admin, lecturer, student accounts)
3. Hardware devices registered (ESP32_CAM_001, RFID_READER_001)

```

**Python AI Service:**
```
1. Flask server starts on port 5001
2. Face recognition models loaded
3. Firebase connection established
4. OpenCV and dlib libraries initialized
5. Face encoding database prepared
```

**Hardware Devices:**
```
ESP32-CAM:
1. WiFi connection established
2. Camera initialized (640x480 resolution)
3. Heartbeat sent to backend every 30 seconds
4. Face detection algorithm activated
5. Status LED indicates ready state

NodeMCU RFID:
1. WiFi connection established
2. RFID reader (MFRC522) initialized
3. LCD display shows "Ready to scan"
4. Heartbeat sent to backend every 30 seconds
5. Offline storage prepared in EEPROM
```

### 2. STUDENT ENROLLMENT PROCESS

**Face Registration:**
```
Admin/Lecturer Portal:
1. Navigate to "Students" → "Add Student"
2. Fill student details (ID, name, email, enrollment year)
3. Assign RFID card number
4. Click "Capture Face" → Camera opens
5. System captures 5-10 face angles
6. Face encodings generated and stored
7. Student profile created with face embeddings
```

**RFID Card Assignment:**
```
1. Physical RFID card distributed to student
2. Card UID registered in student profile
3. Card tested with RFID reader for confirmation
4. Student added to class enrollment list
```

### 3. DAILY ATTENDANCE WORKFLOW

**Method 1: Face Recognition (Primary)**
```
Automatic Process:
1. Student walks in front of ESP32-CAM
2. Camera captures image every 5 seconds
3. Image sent to Python AI service (base64 encoded)
4. Face detection and encoding performed
5. Comparison with enrolled faces database
6. If match found (>85% confidence):
   - Attendance marked as "present"
   - Real-time update sent to frontend
   - Success LED blinks on ESP32-CAM
7. If no match: No action taken
8. Data synced to main backend database
```

**Method 2: RFID Backup (Secondary)**
```
Manual Process:
1. Student taps RFID card on reader
2. Card UID extracted and processed
3. Student lookup in database by RFID card
4. If registered student found:
   - Attendance marked as "present"
   - LCD shows "Attendance OK - [Student Name]"
   - Green LED and success beep
5. If not registered:
   - LCD shows "Card Not Registered"
   - Red LED and error beep
6. Offline mode: Data stored in EEPROM for later sync
```

### 4. REAL-TIME DASHBOARD UPDATES

**Admin Dashboard:**

Real-time Display:

**Lecturer Dashboard:**

**Student Dashboard:**
```
Personal View:

```

### 5. HARDWARE MONITORING SYSTEM

**Device Status Tracking:**
```
Heartbeat Mechanism:
1. Each device sends heartbeat every 30 seconds
2. Backend updates device status and timestamp
3. Frontend shows real-time device status
4. Alerts triggered if device offline >5 minutes
5. Status colors: Green (online), Red (offline), Yellow (maintenance)
```

**Offline Handling:**
```
ESP32-CAM Offline:
- Attendance marking disabled
- System falls back to RFID only
- Admin notified of camera offline

RFID Reader Offline:
- Stores attendance in local EEPROM
- Syncs data when connection restored
- LCD shows "Offline Mode"
- Statistics maintained locally
```

**Backend Issues:**
```
Port 5000 already in use:
→ taskkill /f /im node.exe
→ npm run dev

Authentication errors:
→ Check demo credentials
→ Clear browser localStorage
→ Restart backend service

Database connection failed:
→ Verify DATABASE_URL in .env
→ Check PostgreSQL service running
→ Use in-memory storage for demo
```

**Python AI Service Issues:**
```
Face recognition import error:
→ pip install --upgrade face_recognition
→ Install Microsoft Visual C++ Redistributable
→ pip install cmake dlib

Flask CORS errors:
→ pip install flask-cors
→ Verify backend URL in configuration

Camera connection failed:
→ Check ESP32-CAM power supply
→ Verify WiFi credentials
→ Check server IP address in Arduino code
```

**Hardware Issues:**
```
ESP32-CAM not responding:
→ Check power supply (5V recommended)
→ Verify WiFi connection
→ Check server URL in code
→ Monitor serial output for errors

RFID reader not working:
→ Check SPI connections
→ Verify MFRC522 wiring
→ Test with known RFID cards
→ Check LCD display for feedback

WiFi connection problems:
→ Verify SSID and password
→ Check signal strength
→ Use 2.4GHz network only
→ Monitor serial debug output
```
